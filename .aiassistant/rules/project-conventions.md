# Project conventions

- Treat `AGENTS.md` as the binding instruction file for coding agents.
- Read `config/ai-project.yaml` before substantial work.
- For behavioral or multi-file changes, follow `docs/ai/WORKFLOW.md`.
- Keep active plans and independently verifiable tasks in the temporary work directory referenced by `docs/ai/CURRENT_PLAN.md`.
- Do not invent commands, architecture, or conventions when the repository defines them.
- Keep changes scoped, tested, secure, and documented.
- Run `./scripts/verify.sh` before claiming completion.
- Never expose secrets, credentials, production data, or private keys.
